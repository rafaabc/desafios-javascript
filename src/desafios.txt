Se no if a comparação é com ==, na negação do if é != ?

27/02
Sim, exatamente!

A negação de == é !=

Exemplos:

javascript
// Comparação de igualdade
if (turma == "A") {
    // código se turma for igual a "A"
}

// Negação - diferente de
if (turma != "A") {
    // código se turma for diferente de "A"
}
Outros operadores e suas negações:

== (igual) ↔️ != (diferente)
> (maior) ↔️ <= (menor ou igual)
< (menor) ↔️ >= (maior ou igual)
>= (maior ou igual) ↔️ < (menor)
<= (menor ou igual) ↔️ > (maior)
Então sim, sempre que você quiser verificar se algo é diferente, use !=! 👍

if (turma != "A" || turma != "B" || turma != "C") {
    throw new Error("A turma não pode ser diferente de A, B ou C");
  }
27/02
Cuidado! Essa condição está incorreta e vai sempre ser verdadeira! 🚨

O problema:

Quando você usa || (OU) com !=, a condição sempre será verdadeira porque:

Se turma = "A", então turma != "B" é verdadeiro ✓
Se turma = "B", então turma != "A" é verdadeiro ✓
Se turma = "C", então turma != "A" é verdadeiro ✓
Ou seja, qualquer valor vai entrar no erro!

Solução correta:

Use && (E) ao invés de || (OU):

javascript
if (turma != "A" && turma != "B" && turma != "C") {
    throw new Error("A turma deve ser A, B ou C");
}
Agora sim: só vai dar erro se a turma for diferente de A E diferente de B E diferente de C ao mesmo tempo.

Outra forma (talvez mais fácil de entender):

javascript
if (turma == "A" || turma == "B" || turma == "C") {
    // turma válida, continua o código
} else {
    throw new Error("A turma deve ser A, B ou C");
}
Pense assim: você quer aceitar se for A OU B OU C. Se não for nenhuma dessas, aí sim dá erro! 💡

  const contatosComFormatoDiferente = [];
